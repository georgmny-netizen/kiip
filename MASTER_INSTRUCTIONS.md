## ФАЗА 0 — Подготовка проекта

### 0.1 Создать папку нового проекта
Скопировать `final260426/` в новую папку (например `new-site/`).  
Это базовый шаблон — меняем только контент и конфиг, React-код не переписываем.

```powershell
Copy-Item -Path 'D:\KIIP\site\final260426' -Destination 'D:\KIIP\site\new-site' -Recurse
```

### 0.2 Конвертировать PDF в PNG
Использовать скрипт `ocr_textbook.ps1` для извлечения страниц из PDF.  
Целевая папка для сканов: `D:\KIIP\site\<project>-sources-png\img\`

Изначально файлы получаются как `page-001.png`, `page-002.png` и т.д.

### 0.3 Соглашение об именовании сканов

Итоговый формат после переименования:

| Тип файла | Формат имени | Пример |
|-----------|-------------|--------|
| Страница главы | `tb-NNN-chXX-pY.png` | `tb-079-ch13-p1.png` |
| Страница рабочей тетради | `wb-NNN.png` | `wb-013.png` |
| Разделитель части | `tb-pN-div.png` | `tb-p3-div.png` |
| Обзор части | `tb-pN-review.png` | `tb-p3-review.png` |
| Тест части | `tb-pN-test.png` | `tb-p3-test.png` |
| Обложка | `tb-front-cover.png` | — |

Где `NNN` — сквозной номер страницы (из PDF), `XX` — номер главы, `Y` — страница внутри главы (1–4).

### 1.2 Структура страниц одной главы (стандарт)

Каждая глава занимает **4 страницы**:
1. **p1** — вводная: название главы, `생각해 봅시다`, `학습목표`, `관련 단원 확인하기`
2. **p2** — раздел `01`: основной текст + возможно `알아두면 좋아요`
3. **p3** — раздел `02`: основной текст + возможно `알아두면 좋아요`
4. **p4** — `주요 내용정리` + `이야기 나누기`

### 1.3 Специальные страницы между частями

После последней главы каждой части идут страницы **не добавлять в главы**:
- `대단원 정리` — обобщение части
- `가로 세로 퀴즈` — кроссворд
- `단원 종합 평가` — итоговый тест
- Доп. чтения, разделители

Их нужно оформлять как отдельные **обзоры частей** (`partReviews`).

### 4.3 Источники полей

| Поле | Откуда брать |
|------|-------------|
| `title`, `intro.*`, `objectives`, `related` | p1 — вводная страница |
| `sections[].title`, `sections[].blocks[]` | p2 (раздел 01) и p3 (раздел 02) |
| `didYouKnow` | блок `알아두면 좋아요` на p2/p3 |
| `summary` | блок `주요 내용정리` на p4 |
| `storyBox` | блок `이야기 나누기` на p4 |
| `quiz` | **рабочая тетрадь** `wb-NNN.png` (если есть) ИЛИ ключевые факты из p2–p4 |
| `flashcards`, `interview` | составить вручную по содержанию главы |
| **Все поля `.ru`** | **перевести** корейский текст |

### 4.4 Источники для quiz (если нет рабочей тетради)

1. Вопросы из `알아두면 좋아요`

Если есть рабочая тетрадь — **правильные ответы сверять с ключами ответов** (последние страницы учебника/тетради), не угадывать.

## ФАЗА 5 — Подключение глав в index.js

Файл `src/data/chapters/index.js` использует **lazy loading**:

```js
const loaders = {
  1:  () => import('./chapter-01.js'),
  2:  () => import('./chapter-02.js'),
  // ... добавить запись для каждой созданной главы
  20: () => import('./chapter-20.js'),
};

export const getChapter = async (n) => {
  if (cache[n]) return cache[n];
  const loader = loaders[n];
  if (!loader) return null;
  const mod = await loader();
  cache[n] = mod.default;
  return cache[n];
};
```

После добавления записи в `loaders` глава появится на сайте.

## ⚠️ Правила работы с AI (лимит токенов)

- Заполнять **строго по одной главе за раз** — дождаться подтверждения, потом следующая
- Читать изображения страниц не более **3–4 PNG за один запрос**
- Если файл оборвался — попросить AI **продолжить с места обрыва**, не начинать заново
- Для больших проектов (50+ глав) работать **по разделам**: сначала все главы 편1, потом 편2, и т.д.

## Стиль контента

- Корейский текст — близко к оригинальному тексту учебника
- Русский перевод — понятный, учебный, не слишком вольный
- Не смешивать обзор раздела с последней главой раздела
- Для каждой главы обязательно: `quiz`, `flashcards`, `interview`
- Для всех вариантов ответов обязательно добавлять `ruOptions`
- Поля `didYouKnow` и `storyBox` — опциональны, пропускать если в главе их нет

## Функциональность экзамена (реализовано в шаблоне)

Проект `final260426` включает компонент `ExamSimulation` с двумя режимами:

### Режим «Экзамен» (시험 모드)
- Таймер обратного отсчёта (обычно 30 минут)
- **Кнопка «Завершить экзамен»** — можно закончить в любой момент
- При истечении времени экзамен **не прерывается** — таймер уходит в минус, продолжать можно сколько угодно
- Результаты показываются только после явного завершения

### Режим «Тетрадь» (익힘책 모드)
- **Без таймера** — работа в своём темпе
- **Мгновенная обратная связь** — сразу видно, правильно ли выбран ответ
- **Пояснения** — для каждого варианта показывается:
  - Если ответ правильный — почему он верный
  - Если ответ неправильный — что именно было не так и какой правильный ответ

### Навигация в экзамене
- Переход между вопросами через пагинацию
- Возможность отмечать вопросы для повторного просмотра
- Итоговый экран с результатами: количество правильных/неправильных, процент

### Данные для экзамена
Вопросы для режима экзамена берутся из поля `quiz` каждой главы (см. раздел 4.2). Для полноценного экзамена рекомендуется создать отдельный набор вопросов в `ExamSimulation.jsx` или агрегировать `quiz` из нескольких глав.
